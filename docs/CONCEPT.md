# CONCEPT — プロジェクトの目的と背景

## プロジェクト名

**ai-web-tester** — AIによる自然言語Webテスト実行ツール

## 概要

サイトURLとテスト内容を自然言語で入力すると、AIがブラウザを操作してテストを実行し、スクリーンショット付きのレポートを生成するツール。[taskp](https://github.com/takemo101/taskp) のスキルとして動作する。

## 背景・動機

### 従来のE2Eテストの課題

1. **CSSセレクタの脆弱性** — UI変更のたびにテストが壊れる
2. **テスト作成コストが高い** — Playwright/Cypress のコードを書ける人材が必要
3. **テスト内容が非技術者に読めない** — `page.click('.btn-xyz-123')` は何をしているかわからない

### 解決アプローチ

自然言語でテスト手順を記述し、Vision LLM がスクリーンショットから要素を認識して操作する。

```
従来: page.click('button[data-testid="submit"]')
  ↓
本ツール: 「送信ボタンをクリックする」
```

## ユースケース

### 1. 手動テストの自動化

QAエンジニアが手動で行っているテストシナリオを自然言語で記述し、自動実行する。

```
テスト: ログイン → 商品検索 → カート追加 → 購入確認
期待結果: 購入完了画面が表示されること
```

### 2. デプロイ後の疎通確認

デプロイ後に主要機能が正常に動作するかを自動でチェックする。

```
テスト: トップページが表示される → ナビゲーションリンクが機能する → フォーム送信が成功する
```

### 3. 定期的なサイト監視

cronやCI/CDと組み合わせて、定期的にサイトの動作を検証する。

## 対象ユーザー

- QAエンジニア（テスト自動化の効率化）
- 開発者（デプロイ後の疎通確認）
- プロジェクトマネージャー（テストケースを自然言語で記述・レビュー）

## 技術的な前提

### 既存ツールの活用

本ツールはゼロから作るのではなく、以下の成熟したOSSを組み合わせて構築する。

| コンポーネント | 担当するOSS | 役割 |
|---------------|-------------|------|
| スキル実行基盤 | [taskp](https://github.com/takemo101/taskp) | 入力収集・LLM連携・実行管理 |
| ブラウザ自動化 | [Midscene.js](https://github.com/web-infra-dev/midscene) | Vision LLMによるUI操作 |
| ブラウザエンジン | [Playwright](https://playwright.dev/) | ブラウザ制御・スクリーンショット |

### 調査した類似ツールとの差別化

| ツール | 特徴 | 本ツールとの違い |
|--------|------|------------------|
| [sitegeist](https://github.com/badlogic/sitegeist) | Chrome拡張でAIチャット対話式操作 | 対話型で定時実行不可、Chrome拡張必須 |
| [qa-use](https://github.com/browser-use/qa-use) | Web UIでテスト管理 | クラウドAPIキー必須、ローカルLLM非対応 |
| [Skyvern](https://github.com/Skyvern-AI/skyvern) | Python + Vision LLM | 重量級（Docker + PostgreSQL） |

**本ツールの差別化ポイント:**

1. **taskp スキルとして動作** — 既存の taskp エコシステムに統合
2. **ローカルLLM対応** — Ollama で完全無料実行可能
3. **CLIファースト** — CI/CD・cron との統合が容易
4. **軽量** — npm install だけで動く
