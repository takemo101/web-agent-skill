# CONCEPT — プロジェクトの目的と背景

## プロジェクト名

**web-agent-skill** — AIによる汎用ブラウザ操作エージェント（taskp スキル）

## 概要

自然言語で「やりたいこと」を指示すると、AIがブラウザを自律操作し、スクリーンショット撮影やデータ取得を行うツール。[taskp](https://github.com/takemo101/taskp) のスキルとして動作する。

LLM はスクリプトを生成するのではなく、`curl --json` で操作を1つずつ REPL サーバーに送る。各操作のレスポンスを確認しながら次の動作を決めるため、失敗への対応がリアルタイムで行われる。

## 背景・動機

### Webブラウザ操作の自動化ニーズ

日常的に手動で行っているWeb操作は多い:

- 管理画面のデータ確認・スクショ取得
- 定期的なサイト巡回・情報収集
- SNSやサービスへの投稿・操作
- フォーム入力・申請作業

これらは「やることは決まっているが、毎回手動でやるのが面倒」な作業。

### 従来の自動化の課題

| 方式 | 課題 |
|------|------|
| Selenium / Playwright スクリプト | CSSセレクタが脆弱、UI変更で壊れる |
| RPA ツール | 高価、設定が複雑、環境依存 |
| ブラウザ拡張 | 手動操作が前提、定時実行できない |
| API 直接呼び出し | API がないサービスでは使えない |

### 解決アプローチ

LLM が REPL サーバーに curl で1操作ずつ送り、DOM 直接アクセスで要素を特定して操作する。セレクタではなく意味的な記述（「投稿ボタン」「メールアドレス欄」）で要素を指定するため、UI変更に強い。

```
従来: page.click('button[data-testid="submit"]')  ← UI変更で壊れる
本ツール: clickButton("送信")                      ← ラベルで判断、壊れにくい
```

操作後のレスポンスを即座に確認できるため、失敗した場合は次の curl で `observe` してから description を修正して再試行できる。

## ユースケース

### 1. 定期的な情報収集

```
やること: テック系ニュースサイトの注目記事トップ3のタイトルとURLを取得してスクショを撮る
```

cron で毎朝実行し、チームに共有。

### 2. 管理画面の定期監視

```
やること: ダッシュボードにログインして今月の売上データを取得しスクショを撮る
```

デプロイ後の疎通確認やKPI監視に活用。

### 3. 定型的なWeb操作の自動化

```
やること: 申請フォームに必要事項を入力して送信する
```

毎月・毎週の定型作業を自動化。

### 4. SNS・サービスへの投稿

```
やること: ブログの管理画面にログインし、下書き記事を公開する
```

SNS の自動操作は利用規約を確認すること。

### 5. スクリーンショットによる記録・エビデンス

```
やること: 設定画面を開いて現在の設定内容をスクショする
```

設定変更前の状態記録やエビデンス取得。

## 対象ユーザー

- 開発者（定型作業の自動化、デプロイ後の確認）
- 運用エンジニア（監視、データ取得、定期レポート）
- 個人ユーザー（情報収集、記録取得の自動化）

## 技術的な前提

### 既存ツールの活用

| コンポーネント | 担当するOSS | 役割 |
|---------------|-------------|------|
| スキル実行基盤 | [taskp](https://github.com/takemo101/taskp) | 入力収集・LLM連携・実行管理 |
| REPL サーバー | src/repl-server.ts | LLM から curl で受け取り Playwright で実行 |
| ブラウザ接続 | [Playwright](https://playwright.dev/) CDP | 既存ChromeへのCDP接続・DOM操作 |
| ヘルパーAPI | createAgent(page) | 自然言語記述によるDOM要素特定 |

### 調査した関連ツール

| ツール | 特徴 | 本ツールとの違い |
|--------|------|------------------|
| [sitegeist](https://github.com/badlogic/sitegeist) | Chrome拡張でAIチャット対話式操作 | 対話型で1操作ずつ実行する点は近い。ただし CLI から定時実行できない |
| [browser-use](https://github.com/browser-use/browser-use) | Python + LLM ブラウザエージェント | Python 環境必須、taskp 統合なし |
| [Skyvern](https://github.com/Skyvern-AI/skyvern) | Vision + LLM ワークフロー自動化 | 重量級（Docker + PostgreSQL） |

**本ツールの差別化ポイント:**

1. **1操作ずつ逐次実行** — curl のレスポンスを確認しながら次の操作を決定。失敗に即応できる
2. **セッション管理** — 複数セッションを同時実行。各セッションで独立したタブを管理
3. **ヒストリとリプレイ** — 成功したセッションを JSON で保存し、`replay` で再実行できる
4. **taskp スキルとして動作** — 既存の taskp エコシステムに統合
5. **ローカルLLM対応** — Ollama で完全無料実行可能（Vision LLM不要）
6. **CLIファースト** — cron / CI との統合が容易
7. **既存Chromeを使用** — ログイン状態を維持、storageState管理が不要
