# CONCEPT — プロジェクトの目的と背景

## プロジェクト名

**web-agent** — AIによる汎用ブラウザ操作エージェント

## 概要

自然言語で「やりたいこと」を指示すると、AIがブラウザを自律操作し、スクリーンショット撮影やデータ取得を行い、完了後に指定コマンドを実行するツール。[taskp](https://github.com/takemo101/taskp) のスキルとして動作する。

## 背景・動機

### Webブラウザ操作の自動化ニーズ

日常的に手動で行っているWeb操作は多い：

- 管理画面のデータ確認・スクショ取得
- 定期的なサイト巡回・情報収集
- SNSやサービスへの投稿・操作
- フォーム入力・申請作業

これらは「やることは決まっているが、毎回手動でやるのが面倒」な作業。

### 従来の自動化の課題

| 方式 | 課題 |
|------|------|
| Selenium / Playwright スクリプト | CSSセレクタが脆弱、UI変更で壊れる |
| RPA ツール | 高価、設定が複雑、環境依存 |
| ブラウザ拡張 | 手動操作が前提、定時実行できない |
| API 直接呼び出し | API がないサービスでは使えない |

### 解決アプローチ

Vision LLM がスクリーンショットを見てページを理解し、自然言語の指示に従って自律的に操作する。

```
従来: page.click('button[data-testid="submit"]')  ← UI変更で壊れる
本ツール: 「送信ボタンをクリックする」               ← 見た目で判断、壊れにくい
```

## ユースケース

### 1. 定期的な情報収集

```
やること: テック系ニュースサイトの注目記事トップ3のタイトルとURLを取得してスクショを撮る
完了後: slack-notify.sh に結果を渡す
```

cron で毎朝実行し、チームに共有。

### 2. 管理画面の定期監視

```
やること: ダッシュボードにログインして今月の売上データを取得しスクショを撮る
完了後: python analyze.py --data "{{result}}"
```

デプロイ後の疎通確認やKPI監視に活用。

### 3. 定型的なWeb操作の自動化

```
やること: 申請フォームに必要事項を入力して送信する
完了後: echo "申請完了: {{__timestamp__}}" >> log/submissions.txt
```

毎月・毎週の定型作業を自動化。

### 4. SNS・サービスへの投稿

```
やること: ブログの管理画面にログインし、下書き記事を公開する
完了後: curl -X POST https://hooks.slack.com/... -d '{"text":"記事公開完了"}'
```

※ SNS の自動操作は利用規約を確認すること。

### 5. スクリーンショットによる記録・エビデンス

```
やること: 設定画面を開いて現在の設定内容をスクショする
完了後: cp results/screenshots/*.png ~/Dropbox/evidence/
```

設定変更前の状態記録やエビデンス取得。

## 対象ユーザー

- 開発者（定型作業の自動化、デプロイ後の確認）
- 運用エンジニア（監視、データ取得、定期レポート）
- 個人ユーザー（情報収集、記録取得の自動化）

## 技術的な前提

### 既存ツールの活用

| コンポーネント | 担当するOSS | 役割 |
|---------------|-------------|------|
| スキル実行基盤 | [taskp](https://github.com/takemo101/taskp) | 入力収集・LLM連携・実行管理 |
| ブラウザ自動化 | [Midscene.js](https://github.com/web-infra-dev/midscene) | Vision LLM による自律的UI操作 |
| ブラウザエンジン | [Playwright](https://playwright.dev/) | ブラウザ制御・スクリーンショット |

### 調査した関連ツール

| ツール | 特徴 | 本ツールとの違い |
|--------|------|------------------|
| [sitegeist](https://github.com/badlogic/sitegeist) | Chrome拡張でAIチャット対話式操作 | 対話型、会話コンテキスト維持、定時実行不可 |
| [browser-use](https://github.com/browser-use/browser-use) | Python + LLM ブラウザエージェント | Python 環境必須、taskp 統合なし |
| [Skyvern](https://github.com/Skyvern-AI/skyvern) | Vision + LLM ワークフロー自動化 | 重量級（Docker + PostgreSQL） |

**本ツールの差別化ポイント:**

1. **taskp スキルとして動作** — 既存の taskp エコシステムに統合
2. **完了後コマンド実行** — 操作結果を後続処理に連携
3. **ローカルLLM対応** — Ollama で完全無料実行可能
4. **CLIファースト** — cron / CI との統合が容易
