## LLM の2層構造

本ツールではLLMが2つの異なる役割で使われる。

```
第1層: taskp Agent（スクリプト生成）
  役割: ユーザーの自然言語指示 → Midscene API コードに変換
  入力: 操作指示テキスト + Midscene API仕様
  出力: TypeScript 操作スクリプト
  モデル: テキスト生成モデル（Claude, GPT, Qwen 等）

第2層: Midscene Vision LLM（ブラウザ操作）
  役割: スクリーンショットからページを理解し、自律的に操作
  入力: ページのスクリーンショット + 操作指示
  出力: 操作アクション（クリック座標、入力内容等）
  モデル: Vision モデル（Qwen2.5-VL, GPT-4o, Gemini 等）
```

### 自律操作の仕組み（第2層）

Midscene の `aiAct()` は内部でループを回して自律的にステップを実行する。

```
aiAct("記事ページを開いて、コメント欄に感想を書いて投稿する")
  ↓
  1. 📸 スクリーンショット撮影
  2. 🧠 Vision LLM が画面を分析「記事リンクが見える、クリックしよう」
  3. 🖱️ クリック実行
  4. 📸 スクリーンショット撮影
  5. 🧠 Vision LLM「記事が表示された、コメント欄を探そう」
  6. ⌨️ コメント欄に入力
  7. 📸 スクリーンショット撮影
  8. 🧠 Vision LLM「投稿ボタンが見える、クリックしよう」
  9. 🖱️ クリック実行
  10. 🧠 Vision LLM「投稿完了メッセージが表示された、タスク完了」
  → 終了（最大20サイクル）
```

**毎回スクリーンショットを撮って判断する**ので、ページ構造を事前に知らなくても動作する。

## 処理フロー詳細

### Phase 1: 入力収集（taskp）

```
1. taskp が SKILL.md のフロントマターを解析
2. inputs 定義に従い TUI でユーザーに質問
   - url: 操作対象のURL
   - task: やりたいこと（自然言語、複数行）
   - after_command: 完了後に実行するコマンド（省略可）
   - headless: ヘッドレスモード切替
   - bridge_mode: Bridge Mode 切替
3. 変数を展開し、SKILL.md の本文をプロンプトとして準備
```

### Phase 2: スクリプト生成（taskp Agent）

```
1. taskp の agent モードが LLM を呼び出し
2. LLM が操作指示を解釈し、Midscene API を使ったスクリプトを生成
3. bash ツールで .taskp-tmp/agent-run.ts にスクリプトを書き出し
4. bash ツールで bun run .taskp-tmp/agent-run.ts を実行
```

### Phase 3: ブラウザ操作（Midscene + Playwright）

```
1. Playwright が Chromium を起動（ヘッドレス or headed）
2. ログイン状態を復元（storageState がある場合）
3. 指定URLに遷移
4. PlaywrightAgent を初期化
5. aiAct() で自律操作:
   a. スクリーンショットを撮影
   b. Vision LLM に送信してページを理解
   c. 次のアクションを決定・実行
   d. 完了するまで繰り返し
6. 途中・最終のスクリーンショットを保存
7. データ抽出結果を出力
```

### Bridge Mode での処理フロー差分

| フェーズ | 通常モード | Bridge Mode |
|---------|-----------|-------------|
| ブラウザ起動 | Playwright が Chromium を起動 | 既存の Chrome に接続（Chrome 拡張経由） |
| 認証 | storageState で復元 | 不要（Chrome のセッションを使用） |
| ページ遷移 | `page.goto(url)` | `agent.connectNewTabWithUrl(url)` |
| スクリーンショット | `page.screenshot()` + Midscene レポート | Midscene レポートのみ |
| 終了 | `browser.close()` | `agent.destroy()` |

### Phase 4: 完了後処理

```
1. 最終スクリーンショットを results/screenshots/ に保存
2. HTMLレポートのパスを出力
3. after_command が指定されている場合は実行
   - 抽出データを環境変数やパイプで渡す
4. ターミナルに結果サマリを出力

Bridge Mode では最終スクリーンショットは保存せず、Midscene HTMLレポートを出力する。
```

## 技術選定

| 要素 | 選定 | バージョン | 理由 |
|------|------|-----------|------|
| スキル実行基盤 | taskp | 最新 | 入力収集・LLM連携・実行管理が組み込み済み |
| ブラウザ自動化 | Midscene.js | 1.6.0 | Vision LLM による自律操作、レポート自動生成 |
| ブラウザエンジン | Playwright | 1.58.2 | ヘッドレス対応、storageState、安定性 |
| ランタイム | Bun | >= 1.2.0 | taskp と同一、TS直接実行、ビルド不要 |
| lint + format | Biome | 2.4.9 | Rust製で高速、1ツールで完結 |
| 型チェック | TypeScript | 6.0.2 | `--noEmit` で型チェックのみ |
| テスト | Vitest | 4.1.2 | 高速、Vite エコシステム互換 |
| LLM（スクリプト生成） | Claude / GPT / Ollama | — | taskp の agent モードが対応 |
| LLM（Vision） | Qwen2.5-VL / GPT-4o / Gemini | — | Midscene が対応、ローカルLLM可 |

## エラーハンドリング

| エラー種別 | 原因 | 対処 |
|-----------|------|------|
| 要素が見つからない | Vision LLM が要素を認識できない | エラーメッセージ + スクリーンショット記録 |
| タイムアウト | ページ遷移やロードが遅い | aiWaitFor でタイムアウト設定 |
| ブラウザ起動失敗 | Chromium 未インストール | セットアップ手順で案内 |
| LLM 接続失敗 | APIキー未設定 / サーバーダウン | エラーメッセージで設定方法を案内 |
| ログイン失敗 | Cookie 期限切れ / 2FA | storageState の再取得を案内 |
| 完了後コマンド失敗 | コマンドのパスやパーミッション | エラー出力 + 非ゼロ終了コード |
